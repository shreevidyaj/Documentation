Release Process Summary
=============================================

**THIS PAGE IS CURRENTLY SERVING AS A PLACEHOLDER.  IN PROGRESS for updates.**

This process describes how the development Git repositories are converted into internal Git release repositories and uploaded to the cloud based Git release repositories.

|

The Build & Configuration Lead is responsible for this process. 

|

+----------------------------+---------------------------------------------------------------------------+
|**Entry Criteria/Inputs:**  | - Development repositories ready for release build                        |
|                            | - Documentation repositories ready for release build                      |
|                            |                                                                           |
+----------------------------+---------------------------------------------------------------------------+
|**Exit Criteria/Outputs:**  | - Release build on internal release repositories                          |
|                            | - Labeled baseline on development repositories                            |
|                            | - Release build is uploaded to release server (e.g., WindShare)           |
+----------------------------+---------------------------------------------------------------------------+
