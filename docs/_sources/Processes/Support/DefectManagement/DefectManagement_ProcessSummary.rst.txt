==========================================
Defect Management Process Summary
==========================================

**THIS PAGE IS CURRENTLY SERVING AS A PLACEHOLDER.  IN PROGRESS for updates.**

This process describes the steps in managing defect reports from submission through resolution.

|

The EPM is responsible for ensuring that this process is followed. The Bug Board is responsible for reviewing the defect record and deciding on the bug disposition.

|

+------------------------+---------------------------------------------------------------------------+
|Entry Criteria/Inputs:  | Defecct has been found                                                    |
+------------------------+---------------------------------------------------------------------------+
|Exit Criteria/Outputs:  | - Defect entered/updated in Defect Management System                      |
|                        | - One of the following happens to the defect:                             |
|                        |   - Defect has been fixed                                                 |
|                        |   - Defect is marked Won't Fix                                            |
|                        |   - Defect is determined not to be a defect and Rejected or Withdrawn     |
|                        |   - Defect is determined to be a requirement and reclassified as an       |
|                        |     Enhancement Request                                                   |
+------------------------+---------------------------------------------------------------------------+
